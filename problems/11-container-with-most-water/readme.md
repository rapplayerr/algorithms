# Задача [11]: [container-with-most-water]

**Дата:** 2026-03-07
**Сложность:** Medium
**Паттерн:** two-pointers

## Концепция
уровень воды определяется нижней стенкой, соответсвенно двигая высокую стенку у нас нет шансов увеличить объем: уровень воды не поднимется выше нижней стенки, а длина будет константно уменьшаться. значит, для поиска бОльшего объема имеет смысл двигать меньшую из двух стенок, в надежде найти ту, что будет выше предыдущей и сможет поднять уровень воды до прошлого максимума.

## Ключевая идея
двигаем меньшую из двух стенок, это единственный способ найти бОльший объем.

## Пошаговый разбор
1. запускаем цикл поиска, начинаем с обоих концов, двигаем указатели навстречу друг другу.
2. сначала проверяем текущий объем на превосходство установленного, если больше - меняем установленный максимум.
3. определяем, какой из двух указателей ниже, чтобы двигать именно его. важный ньюанс: при равенстве двух стенок можно сдвинуть их обоих.

## Моё решение
```python
class Solution(object):
    def maxArea(self, height):
        """
        :type height: List[int]
        :rtype: int
        """
        left = 0 
        right = len(height) - 1
        maxed = 0
        while left < right:
            vol = min(height[left], height[right])*(right-left)
            if vol>maxed:
                maxed = vol
            if height[left]<height[right]:
                left+=1
            elif height[left]>height[right]:
                right-=1
            else:
                left+=1
                right-=1            
        return maxed
